Using Host Volume

Using Hostpath Volume

https://kubernetes.io/docs/concepts/storage/volumes/#hostpath

背景

和docker一样，k8s的pod在删除后，运行时产生的数据不会保存下来。

kubernetes提供了多种持久化存储的方案，比如ebs、nfs、local：

这一节先介绍最简单的volume，直接使用worker node本身的存储。又叫hostpath

使用hostpath

hostpath的pod定义如下：

apiVersion: v1
kind: Pod
metadata:
  name: test-pd
spec:
  containers:
  - image: nginx
    name: nginx
    volumeMounts:
    - mountPath: /mydata
      name: test-volume
  volumes:
  - name: test-volume
    hostPath:
      # directory location on host
      path: /tmp
      # this field is optional
      type: Directory

这样将宿主机的/tmp目录挂载到容器的/mydata目录。


登录到pod的bash，在容器中新建一个文件：

[@BDSZYF000132741:pod]$ kubectl exec -it test-pd bash
root@test-pd:/# ls
bin boot dev etc home lib lib64 media mnt mydata opt proc root run sbin srv sys tmp usr var
root@test-pd:/# cd /mydata/
root@test-pd:/mydata# ls
systemd-private-4e4ecf17541e49ad9046bc0c82614925-chronyd.service-nDFeL0
root@test-pd:/mydata# echo "Hi">>hello.txt

登录到pod所在的worker节点，在宿主机的/tmp目录下也能看到这个文件：

[root@ip-192-168-175-56 /]# cd tmp
[root@ip-192-168-175-56 tmp]# ls
systemd-private-4e4ecf17541e49ad9046bc0c82614925-chronyd.service-nDFeL0
[root@ip-192-168-175-56 tmp]# ls
systemd-private-4e4ecf17541e49ad9046bc0c82614925-chronyd.service-nDFeL0

[root@ip-192-168-175-56 tmp]# ls
hello.txt systemd-private-4e4ecf17541e49ad9046bc0c82614925-chronyd.service-nDFeL0
[root@ip-192-168-175-56 tmp]# cat hello.txt
Hi


总结

使用hostpath，适合只有一个worker node的简单场景。

如果有多个节点，在不同的worker node上运行多个pod，那么产生的数据会分布到每个节点。


所以，在生产环境中，需要将产生的数据集中保存。
